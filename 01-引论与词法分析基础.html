第一~二讲(1-3节)：引论与词法分析基础

一、知识点讲解

1.1 编译程序概述

1.1.1 什么是编译程序

编译程序（Compiler）是一种能够把某一种语言程序（源语言程序）转换成另一种语言程序（目标语言程序）的程序，后者与前者在逻辑上是等价的。计算机上执行一个高级语言程序一般分为两个步骤：首先，使用编译程序把高级语言翻译成机器语言程序；其次，运行机器语言程序求得计算结果。

实例：Pascal语句 x := a+b 或 C++语句 x = a+b，经编译后生成汇编指令（如 MOV AX, [A]、ADD AX, BX 等），最终转换为十六进制机器代码。

1.1.2 编译器与解释器的区别

特性 编译器 解释器
工作方式 先翻译后执行 边翻译边执行
工作效率 时间快、空间省 时间慢、空间费
交互性与动态特性 差 好
可移植性 差 好
典型代表 C/C++、Pascal 早期Basic、Java（字节码解释）

从翻译的角度来讲，两种方式所涉及的原理、方法、技术是相似的。编译程序根据用途进一步分为：

  • 诊断编译程序（Diagnostic Compiler）：专门用于帮助程序开发和调试
  • 优化编译程序（Optimizing Compiler）：着重于提高目标代码效率
  • 交叉编译程序（Cross Compiler）：产生不同于宿主机的机器代码
  • 可变目标编译程序（Retargetable Compiler）：不需重新编译与机器无关的部分就能改变目标机

1.1.3 程序设计语言的分类

按范型划分：

  • 过程式语言：FORTRAN、C、Pascal
  • 面向对象语言：C++、Java、Ada95
  • 函数式语言：LISP、ML（递归特性，面向特定领域）
  • 逻辑程序设计语言：Prolog
  • 脚本式语言：Shell（仅是一种安排，没有复杂的逻辑关系）
  • 其他面向特定应用领域的语言：HTML/XML（互联网）、MATLAB（计算机辅助设计）、VHDL/Verilog（集成电路设计）

1.1.4 语言翻译的类型

  1. 汇编语言 → 机器指令：汇编（或交叉汇编）
  2. 程序设计语言 → 汇编语言/机器指令：编译（或解释）
  3. 高级语言之间：转换（或预编译）
  4. 逆向：反汇编、反编译

1.1.5 语言的三个维度

  • 语法：语言的结构，即语言的"样子"——用一组规则形成和产生合式的程序（词法规则+语法规则）
  • 语义：附着于语言结构上的实际含义，即语言的"意义"——定义单词符号和语法单位的意义
  • 语用：语言的使用方式

1.2 编译程序的工作原理与基本结构

1.2.1 通用程序设计语言的主要成分

程序由声明性语句（提供操作对象的性质，如数据类型、值、作用域）和操作性语句（确定操作的计算次序，完成实际操作）两部分组成。

编译器的翻译：

  • 声明：生成相应的环境，一般是配置存储空间
  • 操作：生成可执行的代码序列

1.2.2 编译的基本过程（五个阶段）

源程序 → 词法分析 → 语法分析 → 语义分析 → 中间代码生成 → 代码优化 → 目标代码生成 → 目标程序

阶段1：词法分析

  • 输入：源程序字符串
  • 输出：单词符号流（记号流）
  • 任务：从左到右逐个字符读入源程序，识别出一个个单词（关键字、标识符、字面量、特殊符号）
  • 依据：语言的词法规则（构词规则），工具是正规式和有限自动机

阶段2：语法分析

  • 输入：单词符号流
  • 输出：语法树
  • 任务：识别语言结构，以树的形式表示
  • 依据：语法规则（上下文无关文法），理论模型是下推自动机

阶段3：语义分析

  • 任务：检测结构正确的句子是否语义合法，修改树结构
  • 工具：属性文法

阶段4：中间代码生成（可选）

  • 生成一种既接近目标语言，又与具体机器无关的表示，便于优化与代码生成
  • 中间代码形式：三地址码、四元式等
  • 到这一阶段为止，编译器与解释器可以一致

阶段5：目标代码生成

  • 产生不同形式的目标代码：汇编、可重定位、内存形式（Load-and-Go）

辅助模块：

  • 符号表管理：合理组织符号，便于各阶段查找、填写
  • 出错处理：处理词法错、语法错、静态语义错、动态语义错

1.2.3 编译的前端和后端

  • 前端：语言结构和意义的分析（词法分析、语法分析、语义分析）
  • 后端：语言意义处理（中间代码生成、优化、目标代码生成）
  • 中间代码：前端与后端的分界
  • 编译器基础架构（Infrastructure）：源代码的中间表示 + 一组前端 + 一组后端

1.2.4 编译器扫描的遍数

每个阶段将程序完整分析一遍的工作模式称为一遍扫描。确定扫描遍数的因素：

  • 软、硬件条件（如内存太小，或需要全局优化）
  • 语言结构（如规定标识符的先声明后引用）
  • 编译技术（如拉链-回填技术处理 goto 语句）

1.2.5 编译器的编写方式

  1. 直接使用汇编语言和程序设计语言
  2. 利用编译器编写工具：词/语法分析生成器（LEX/YACC）、语法制导翻译工具、自动代码生成器、数据流工具
  3. 基于编译器基础架构的构造系统（开放式编译器，如 GCC、SUIF）

1.3 词法分析基础

1.3.1 记号、模式与单词

三个核心术语：

  • 模式（Pattern）：产生和识别元素的规则
  • 记号（Token）：按照某个模式识别出的元素类别（一组）
  • 单词（Lexeme）：被识别出的元素自身的值（一个），也称为词值

记号的属性： $$\text{记号} = \text{记号的类别} + \text{记号的属性}$$

例如表达式 mycount > 25 由三个记号组成：id(82, "mycount")、relation(81, 5)、num(83, 25)。

单词的基本分类：

类别 缩写 示例
关键字（保留字） kw const、if
标识符 id pi、count、D2
字面量 literal "core dumped"
运算符 — +、-、*、/
分隔符 ks ;、,、(、)

1.3.2 词法分析器的功能

  1. 滤掉源程序中的无用成分（注释、空格、回车等）
  2. 处理与具体平台有关的输入（如文件结束符的不同表示）
  3. 识别记号并交给语法分析器
  4. 调用符号表管理器或出错处理器

工作方式：

  • 单独一遍扫描
  • 作为语法分析器的子程序
  • 并行方式

1.3.3 源程序的输入和预处理

词法分析器的任务是从左到右逐个字符地读入源程序，对构成源程序的字符流进行扫描和分解，识别出一个个单词，把作为字符串的源程序改造成为单词符号串的中间程序。

预处理：去除注释、无用的空白、跳格、回车、换行符等，装入词法分析器的扫描缓冲区。

1.4 形式语言基础

1.4.1 语言的形式化定义

定义2.1（乔姆斯基）：语言 $L$ 是有限字母表 $\Sigma$ 上有限长度字符串的集合。

两个"有限"的含义：

  1. 字母表 $\Sigma$ 是有限的
  2. 字符串的长度是有限的

1.4.2 字符串的基本概念

术语 含义 示例（S="abc"）
$|S|$ 字符串的长度 $|abc| = 3$
$\varepsilon$ 空字符串 $|\varepsilon| = 0$
$S_1S_2$ 连接 "abc""def" = "abcdef"
$S^n$ 幂 $(\text{"abc"})^3 = \text{"abcabcabc"}$
前缀 — $\varepsilon$, a, ab, abc
后缀 — $\varepsilon$, c, bc, abc
子串 — $\varepsilon$, a, b, c, ab, bc, abc
真前缀 非自身前缀 a, ab
真后缀 非自身后缀 c, bc
真子串 非自身子串 a, b, c, ab, bc
子序列 去掉若干不一定连续字符 "abdf"是"abcdef"的子序列

1.4.3 字符串集合的运算

运算 定义 示例（若 $L=\{a,b\}, M=\{c,d\}$）
并 $L \cup M$ $\{s \mid s \in L \text{ 或 } s \in M\}$ $\{a,b,c,d\}$
交 $L \cap M$ $\{s \mid s \in L \text{ 且 } s \in M\}$ $\Phi$ (空)
连接 $LM$ $\{st \mid s \in L \text{ 且 } t \in M\}$ $\{ac,bc,ad,bd\}$
闭包 $L^*$ $L^0 \cup L^1 \cup L^2 \cup \ldots$ $\{\varepsilon, a, b, aa, bb, ab, ba, \ldots\}$
正闭包 $L^+$ $L^1 \cup L^2 \cup L^3 \cup \ldots$ $\{a, b, aa, bb, ab, ba, \ldots\}$

重要示例：令 $L = \{A\text{-}Z, a\text{-}z\}$，$D = \{0\text{-}9\}$

  • $L \cup D$：字母和数字的集合
  • $LD$：一个字母后随一个数字的串的集合
  • $L^4$：四个字母的串的集合
  • $L(L \cup D)^*$：以字母开头的所有字母数字串的集合（即标识符）
  • $D^+$：不含空串的数字串的集合

1.5 正规式与正规集

1.5.1 正规式的定义

定义2.2：令 $\Sigma$ 是一个有限字母表，则 $\Sigma$ 上的正规式及其表示的集合递归定义如下：

  1. $\varepsilon$ 是正规式，表示集合 $L(\varepsilon) = \{\varepsilon\}$
  2. 若 $a$ 是 $\Sigma$ 上的字符，则 $a$ 是正规式，表示集合 $L(a) = \{a\}$
  3. 若正规式 $e_1$ 和 $e_2$ 分别表示集合 $L(e_1)$ 和 $L(e_2)$，则：
  4. (a) $e_1|e_2$ 是正规式，表示 $L(e_1) \cup L(e_2)$
  5. (b) $e_1e_2$ 是正规式，表示 $L(e_1)L(e_2)$
  6. (c) $e_1^*$ 是正规式，表示 $(L(e_1))^*$
  7. (d) $(e_1)$ 是正规式，表示 $L(e_1)$

可用正规式描述的语言称为正规语言或正规集。

1.5.2 运算的优先级与结合性

  1. 三种运算均具有左结合性质
  2. 优先级从高到低：闭包(*) > 连接 > 或(|)

例如：$(a)|(((b)^*)(c))$ 可简化为 $a|b^*c$

1.5.3 正规式的等价

定义：若正规式 $P$ 和 $Q$ 表示了同一个正规集，则称 $P$ 和 $Q$ 是等价的，记为 $P = Q$。

正规式的代数性质（A、B、C为正规式）：

公理 表达式
等价性 $A|B = B|A$
结合性 $(A|B)|C = A|(B|C)$，$(AB)C = A(BC)$
分配律 $A(B|C) = AB|AC$，$(A|B)C = AC|BC$
连接恒等 $\varepsilon A = A\varepsilon = A$
闭包幂等 $A^{**} = A^*$
$\varepsilon$ 吸收 $\varepsilon^* = \varepsilon$

正规式等价的判定方法：

  1. 根据定义，证明不同的正规式表示同一集合
  2. 根据正规式的代数性质进行运算

1.5.4 记号的正规定义

用正规式定义记号的公式为：记号 = 正规式

可读作"（左边）记号定义为（右边）正规式"，或"记号是正规式"。

例如 id = a(a|b)^* 可读作"id定义为 $a(a|b)^*$"。

1.5.5 正规式的缩写形式

(a) 正闭包 +：$r^+ = rr^* = r^*r$，$r^* = r^+|\varepsilon$

(b) 可缺省 ?：$r? = r|\varepsilon$

(c) 字符组 [ ]：

  • 枚举：[abc] 等价于 $a|b|c$
  • 分段：[0-9a-z] 等价于 $0|1|...|9|a|b|...|z$

(d) 非字符组 [^ ]：若 $[r]$ 是一个字符组，则 $[\text{^}r]$ 表示 $\Sigma - L(r)$ 例如：若 $\Sigma = \{a,b,c,d,e,f,g\}$，则 $L([\text{^}abc]) = \{d,e,f,g\}$

(e) 辅助定义式：辅助定义的形式与正规式一样，但它不与任何模式匹配，仅供内部使用。

缩写后的完整示例：

char              = [a-zA-Z]
digit             = [0-9]
digits            = digit+
optional_fraction = ( . digits )?
optional_exponent = ( E (+|-)? digits )?
id  = char ( char|digit )*
num = digits optional_fraction optional_exponent

1.6 有限自动机

1.6.1 有限自动机概述

有限自动机（Finite Automata, FA）是描述特定类型算法的数学方法，可用作描述在输入串中识别模式的过程，因此也能被用作构造词法分析程序。

根据在识别输入单个字符的过程中下一状态是否确定，分为两类：

  • 确定型有限自动机（DFA）
  • 非确定型有限自动机（NFA）

二者都能识别正规集。DFA 的识别器比 NFA 的识别器快得多，但 DFA 可能比等价的 NFA占用更多空间（时空权衡）。

1.6.2 状态转换图

状态转换图是由一组矢量线连接的有限个节点所组成的有向图，是有限自动机的直观图示。一个状态转换图可以用于识别一定的字符串，即从初态出发到某一终态路径上字符的连接。

1.6.3 不确定的有限自动机（NFA）

定义2.4：NFA 是一个五元组 $M = (S, \Sigma, move, s_0, F)$，其中：

  1. $S$：有限个状态的集合
  2. $\Sigma$：有限个输入字符（包括 $\varepsilon$）的集合
  3. $move$：状态转移函数，$move(s_i, ch) = s_j$，表示当前状态 $s_i$ 下若遇到输入字符 $ch$，则转移到状态 $s_j$，且 move 函数是一对多的
  4. $s_0$：唯一的初态（开始状态）
  5. $F$：终态集（接受状态集），$F \subseteq S$

NFA 的三种表示方式：

  1. 形式定义（五元组）
  2. 状态转换图（有向图）：每个状态对应一个节点，每个 $move(s_i, a) = s_j$ 对应一条有向边
  3. 状态转换矩阵（二维数组）：$M[s_i, a]$ 中的内容是从状态 $s_i$ 出发经字符 $a$（或 $\varepsilon$）所到达的下一状态 $s_j$

NFA 的不确定性表现：

  • 定义：move 函数是 1对多的
  • 转换图：同一状态有多于一条边标记相同字符转移到不同的状态
  • 转换矩阵：$M[s_i, a]$ 是一个状态的集合

1.6.4 确定的有限自动机（DFA）

定义2.5：DFA 是 NFA 的一个特例，其中：

  1. 没有状态具有 $\varepsilon$ 状态转移（状态转换图中没有标记 $\varepsilon$ 的边）
  2. 对每个状态 $s$ 和每个字符 $a$，最多有一个下一状态

DFA 的特征（确定性）：

  • 定义：$move(s_i, a)$ 函数是 1对1的
  • 转换图：从一个节点出发的边上标记均不相同
  • 转换矩阵：$M[s_i, a]$ 是一个状态
  • 字母表不包括 $\varepsilon$

1.6.5 DFA 模拟器（算法2.1）

s := s0; ch := nextchar;     -- 初值
while ch ≠ eof               -- 循环
  s := move(s, ch);
  ch := nextchar;
end loop;
if s is in F                  -- 返回
  then return "yes";
  else return "no";
end if;

特点：算法与模式无关，仅 DFA 与模式相关。算法（模拟器）+ DFA（数据）= 词法分析器核心。

1.6.6 NFA 与 DFA 的关系

  • DFA 是 NFA 的特例（对 NFA 施加两条限制即可得到 DFA）
  • 二者识别能力等价：对任意 NFA，存在等价的 DFA
  • DFA 识别速度快但状态多；NFA 状态少但需要回溯
  • 正规式 → NFA（结构对应、算法简单） → DFA（确定化） → 最小化 DFA → 词法分析器

1.6.7 乔姆斯基文法体系

对 0 型文法施加以下第 $i$ 条限制，即得到 $i$ 型文法：

  • 1型（上下文有关文法）：产生式 $\alpha \to \beta$（$S \to \varepsilon$ 除外）满足 $|\alpha| \leq |\beta|$
  • 2型（上下文无关文法）：产生式形如 $A \to \beta$，其中 $A \in N$，$\beta \in (N \cup T)^*$
  • 3型（正规文法）：产生式形如 $A \to a$ 或 $A \to aB$（或 $A \to Ba$），其中 $A,B \in N$，$a \in T$
文法类型 自动机 描述能力
0型（无限制） 图灵机 递归可枚举语言
1型（上下文有关） 线性界限自动机 上下文有关语言
2型（上下文无关） 下推自动机 上下文无关语言
3型（正规） 有限自动机 正规语言

关键结论：词法分析使用 3 型文法（正规式+有限自动机），语法分析使用 2 型文法（上下文无关文法+下推自动机）。

二、相关真题与解析

题目1（来源：2022年 试题A）

原题： 识别2型文法描述语言的自动机是__________。

解析/答案： 下推自动机。2型文法即上下文无关文法，与之对应的自动机模型是下推自动机（Pushdown Automaton, PDA）。0型→图灵机，1型→线性界限自动机，2型→下推自动机，3型→有限自动机。

题目2（来源：2022年 试题A）

原题： 描述含偶数个0和偶数个1的01串的正规式是__________。

解析/答案： 这类问题需通过构造 DFA 后反推正规式。含偶数个0和偶数个1意味着0的个数和1的个数均为偶数。这需要4个状态来表示奇偶组合：(偶0,偶1)、(奇0,偶1)、(偶0,奇1)、(奇0,奇1)。从状态转移关系可推出等价的正规式。

题目3（来源：2022年 试题A）

原题： 一个NFA需要满足__________和__________，才是一个DFA。

解析/答案： （1）没有 $\varepsilon$ 状态转移（$\varepsilon$-transition）；（2）对每个状态和每个输入字符，最多有一个下一状态。这是 DFA 定义中对 NFA 施加的两条限制条件。

题目4（来源：2022年 试题A）

原题： 设有表达式 A+b*C，将其中 C 识别为标识符的编译阶段是__________。

解析/答案： 词法分析阶段。词法分析阶段的任务是从左到右逐个字符地读入源程序，识别出一个个单词（关键字、标识符、字面量、特殊符号等）。将 C 识别为标识符（id）正是词法分析器的工作。

题目5（来源：2022年 试题A）

原题： 编译器可以发现程序中的__________错误、__________错误和__________错误。

解析/答案： 词法错误、语法错误、语义错误（静态语义错误）。编译时可诊断的错误类型包括：词法错误（如非法字符）、语法错误（如少分号、begin/end不配对）、静态语义错误（如类型不一致、参数不匹配）。动态语义错误（如除数为零、无穷递归）在运行时才能发现。

题目6（来源：2022年 试题B）

原题： __________和__________是编译程序各个阶段都涉及到的工作。

解析/答案： 符号表管理和出错处理。这两个模块贯穿编译的各个阶段，每个阶段都可能需要访问符号表或报告错误。

题目7（来源：2022年 试题B）

原题： 描述含101子串的01串的正规式为__________。

解析/答案： $(0|1)^*101(0|1)^*$。该正规式表示在任意01串中包含子串"101"。$(0|1)^*$ 匹配任意前缀和后缀，中间必须出现101。

题目8（来源：2022年 试题B）

原题： 若不考虑声明与引用，程序设计语言中的语法结构可采用__________型文法描述。

解析/答案： 2型（上下文无关文法）。程序设计语言的语法结构（如表达式、语句、程序等）通常用上下文无关文法（2型）描述。1型上下文有关文法用于处理声明与引用等需要上下文的场合。

题目9（来源：2022年 试题B）

原题： 上下文无关文法的组成部分有__________、__________、__________、开始符号。

解析/答案： 非终结符集合（$N$）、终结符集合（$T$）、产生式集合（$P$）。上下文无关文法 $G = (N, T, P, S)$，由这四个部分组成，其中 $S$ 为开始符号。

题目10（来源：2022年 试题A）

原题： 简答题——编译程序中，语义分析和生成中间代码最常用的方法是什么？

解析/答案： 语法制导翻译方法（Syntax-Directed Translation, SDT）。它以语法分析为基础，在语法分析得到语言结构的结果时，对附着于此结构的语义进行处理（如计算表达式的值、生成中间代码等）。目前大多数编译程序普遍采用基于属性文法的语法制导翻译方法。具体来说，就是为每个产生式关联一组语义规则，在语法分析过程中根据所使用的产生式执行相应的语义动作。

题目11（来源：2022年 试题A）

原题： 简答题——正规式 $(a|b|c)?(d|e)$ 描述的正规集中包含哪些元素？

解析/答案： 逐一枚举：

  • $(a|b|c)?$ 表示可选，即可以取 $a$、$b$、$c$ 或 $\varepsilon$（空）
  • $(d|e)$ 表示取 $d$ 或 $e$

组合结果：$d$、$e$、$ad$、$ae$、$bd$、$be$、$cd$、$ce$

共 8 个元素。正规集为 $\{d, e, ad, ae, bd, be, cd, ce\}$。

题目12（来源：2022年 试题B）

原题： 简答题——简述DFA与NFA的区别。

解析/答案： DFA 与 NFA 的主要区别：

  1. 状态转移：DFA 的 move 函数是 1对1 的（每个状态对每个输入字符最多有一个下一状态），NFA 的 move 函数是 1对多 的（一个状态对同一字符可能有多个下一状态）
  2. $\varepsilon$ 转移：DFA 不允许有 $\varepsilon$ 状态转移，NFA 允许有 $\varepsilon$ 状态转移（状态转换图中可标记 $\varepsilon$ 的边）
  3. 识别效率：DFA 识别速度快（不需要回溯），NFA 识别需要试探和回溯
  4. 状态数量：等价的 DFA 通常比 NFA 有更多的状态（子集构造法可能导致状态数指数增长）
  5. 表达能力：二者等价，都识别正规集（正规语言）
