第二讲(4-6节)：词法分析器的构造

一、知识点讲解

1.1 从正规式到词法分析器的总体路线

构造词法分析器的一般方法遵循以下五步过程：

$$\text{正规式} \rightarrow \text{NFA} \rightarrow \text{DFA} \rightarrow \text{最小化DFA} \rightarrow \text{词法分析器}$$

为什么需要经过 NFA 这一步？ - 正规式 → NFA：有规范的一对一的构造算法（Thompson 算法） - DFA → 词法分析器：有便于单词符号识别的算法（DFA 模拟器） - NFA → DFA：有规范的确定化算法（子集构造法） - 最小化 DFA：得到状态数最少的等价 DFA，提高空间效率

1.2 有限自动机的等价性

定义2.6：若有两个有限自动机 $M_1$ 和 $M_2$，满足 $L(M_1) = L(M_2)$，即它们接受相同的语言（识别同一个正规集），则称 $M_1$ 和 $M_2$ 是等价的，记为 $M_1 = M_2$。

定理2.1：设 $L$ 是被 NFA $M_1$ 所接受的语言，那么一定存在一个 DFA $M_2$，满足 $L(M_1) = L(M_2)$。

关键结论：对任意正规式 $e$，均可构造一个 NFA $M$，使 $L(M) = L(e)$。NFA 和 DFA 的描述能力等价。

1.3 Thompson 算法：从正规式构造 NFA

1.3.1 算法描述

算法2.2（Thompson 算法） - 输入：字母表 $\Sigma$ 上的正规式 $r$ - 输出：接受 $L(r)$ 的 NFA $N$ - 方法：首先分解 $r$，然后按下述步骤自下而上构造 NFA

1.3.2 基本构造规则

(1) 对 $\varepsilon$：构造 NFA $N(\varepsilon)$，其中 $s_0$ 为初态，$f$ 为终态，此 NFA 接受 $\{\varepsilon\}$。

$$N(\varepsilon): s_0 \xrightarrow{\varepsilon} f$$

(2) 对字母表上的每个字符 $a$：构造 NFA $N(a)$，接受 $\{a\}$。

$$N(a): s_0 \xrightarrow{a} f$$

(3) 对复合正规式（设 $N(p)$ 和 $N(q)$ 是正规式 $p$ 和 $q$ 的 NFA）：

正规式 NFA 构造方法 接受语言
$p|q$ 新建初态 $s_0$，分别以 $\varepsilon$ 边连接到 $N(p)$ 和 $N(q)$ 的初态；$N(p)$ 和 $N(q)$ 的终态以 $\varepsilon$ 边连接到新的终态 $f$ $L(p) \cup L(q)$
$pq$ $N(p)$ 的初态作为整体初态，$N(p)$ 的终态以 $\varepsilon$ 边连接到 $N(q)$ 的初态，$N(q)$ 的终态作为整体终态 $L(p)L(q)$
$p^*$ 新建初态 $s_0$ 和终态 $f$，$s_0$ 以 $\varepsilon$ 边分别连接到 $N(p)$ 的初态和 $f$，$N(p)$ 的终态以 $\varepsilon$ 边分别连接到 $N(p)$ 的初态和 $f$ $(L(p))^*$

(4) 对于 $(p)$：直接使用 $p$ 本身的 NFA，不再构造新的 NFA。

1.3.3 正规式与 NFA 的对应关系总结

正规式 表示集合
$\varepsilon$ $L(\varepsilon) = \{\varepsilon\}$
$a$ $L(a) = \{a\}$
$P|Q$ $L(P) \cup L(Q)$
$PQ$ $L(P)L(Q)$
$P^*$ $(L(P))^*$
$(r)$ $L(r)$

Thompson 算法的特点： - 构造与正规式一一对应 - 构造一个新的 NFA 最多增加两个状态

1.3.4 构造示例

例2.14：用 Thompson 算法构造正规式 $r = (a|b)^*abb$ 的 NFA $N(r)$

步骤： 1. 分解正规式：$(a|b)^*abb = ((a|b)^*)(a)(b)(b)$ 2. 自下而上构造： - 先构造 $N(a)$ 和 $N(b)$ - 构造 $N(a|b)$（应用 $p|q$ 规则） - 构造 $N((a|b)^*)$（应用 $p^*$ 规则） - 依次连接 $N(a)$、$N(b)$、$N(b)$（应用 $pq$ 规则） 3. 最终得到包含 11 个状态的 NFA

1.4 NFA 的确定化：子集构造法

1.4.1 确定化的思想

NFA 存在不确定性（$\varepsilon$ 转移、同一字符多个下一状态），使得难以用计算机程序直接模拟。子集构造法的核心思想是：让新构造的 DFA 的每个状态代表 NFA 的一个状态集，DFA 用它的状态去记住 NFA 在读输入符号后能到达的所有状态。

1.4.2 关键函数定义

$\varepsilon$-闭包($T$)： 从状态集 $T$ 出发，不经任何字符（仅通过 $\varepsilon$ 边）能到达的状态全体。

形式化定义：状态集 $T$ 的 $\varepsilon$-闭包($T$) 是一个状态集，满足： 1. $T$ 中所有状态属于 $\varepsilon$-闭包($T$) 2. 任何 $smove(\varepsilon\text{-闭包}(T), \varepsilon)$ 属于 $\varepsilon$-闭包($T$) 3. 再无其他状态属于 $\varepsilon$-闭包($T$)

$smove(S, a)$： 从状态集 $S$ 出发，通过标记为 $a$ 的边能到达的下一状态全体。与 $move(s, a)$ 的区别在于用状态集取代单个状态。

1.4.3 NFA 模拟算法（算法2.2'）

S := ε-闭包({s0});           -- 所有可能初态的集合
a := nextchar;
while a ≠ eof loop
    S := ε-闭包(smove(S, a)); -- 所有下一状态的集合
    a := nextchar;
end loop;
if S ∩ F ≠ Φ then return "yes";
else return "no";
end if;

NFA 模拟与 DFA 模拟的三点区别：

区别点 模拟 DFA 模拟 NFA
开始状态 初态 $s_0$（单个） 初态集 $S = \varepsilon\text{-闭包}(\{s_0\})$
下一状态转移 下一状态（单个） 下一状态集（$S := \varepsilon\text{-闭包}(smove(S, a))$）
接受判断 $s \in F$ $S \cap F \neq \Phi$

1.4.4 子集构造法（算法2.3）

输入: NFA N
输出: 等价的 DFA D

ε-闭包({s0}) 是 Dstates 仅有的状态，且尚未标记;
while Dstates 有尚未标记的状态 T loop
    标记 T;
    for 每一个输入字符 a loop
        U := ε-闭包(smove(T, a));
        if U 非空 then
            Dtran[T, a] := U;
            if U 不在 Dstates 中 then
                U 作为尚未标记的状态加入 Dstates;
            end if;
        end if;
    end loop;
end loop;

数据结构：$Dstates$（DFA 状态集）、$Dtran$（DFA 状态转移表）

1.4.5 确定化完整示例

例2.15：构造 $(a|b)^*abb$ 的 DFA

NFA 初态为 0，终态为 10。计算过程：

$$A = \varepsilon\text{-闭包}(\{0\}) = \{0,1,2,4,7\}$$ $$B = \varepsilon\text{-闭包}(smove(A, a)) = \{1,2,3,4,6,7,8\}$$ $$C = \varepsilon\text{-闭包}(smove(A, b)) = \{1,2,4,5,6,7\}$$ $$D = \varepsilon\text{-闭包}(smove(B, b)) = \{1,2,4,5,6,7,9\}$$ $$E = \varepsilon\text{-闭包}(smove(D, b)) = \{1,2,4,5,6,7,10\}$$

DFA 状态转移表：

状态 a b
A B C
B B D
C B C
D B E
E B C

重命名状态：A→0, B→1, C→2, D→3, E→4，其中 4 为终态（包含原 NFA 终态 10）。

1.5 DFA 的最小化

1.5.1 等价状态与可区分状态

定义：设 $p, q \in S$，对于任一输入序列 $w \in \Sigma^*$： - 若 $move(p, w) \in F$ 且 $move(q, w) \in F$，则称状态 $p$ 和 $q$ 等价 - 否则，称 $p$ 和 $q$ 是可区分的——即存在 $x \in \Sigma^*$，使得 $move(p, x)$ 和 $move(q, x)$ 一个为终态一个为非终态

直观理解：如果从 $p$ 和 $q$ 出发分析任何输入序列都得到相同结果（都接受或都不接受），则 $p$ 和 $q$ 可以合并成一个状态。

1.5.2 最小化算法（算法2.4）

1. 初始划分: Π = {S-F, F1, F2, ...} （非终态组和按接受记号分的终态组）

2. 反复分裂: for Π 的每一个组 G loop
       划分 G: G 的两个状态 s 和 t 在同一组的充要条件是:
       ∀a.(move(s,a) ∈ Gi ∧ move(t,a) ∈ Gi)  -- Gi 是 Π 中某个组
       用新划分的组替代 G，形成新的划分 Πnew
   end loop;

3. 若 Πnew = Π，令 Πfinal = Π，转 4；否则令 Π = Πnew 并重复步骤 2

4. 在 Πfinal 每个组 Gi 中选一个代表 si'，修改状态转移使所有进入/离开该组的转移指向/来自代表

5. 删除死状态（非终态且对所有输入字符均转向自身）和不可达状态

1.5.3 最小化示例

例2.17：化简 DFA

状态 a b
A B C
B B D
C B C
D B E
E B C

其中 E 为终态。

  1. 初始划分：$\Pi_1 = \{\{A,B,C,D\}, \{E\}\}$
  2. 迭代分裂：
  3. 考察 $\{A,B,C,D\}$ 中状态对 b 的转移：$m(D,b)=E$（终态组），而 $m(A,b)=C, m(B,b)=D, m(C,b)=C$（均不在终态组）。但 D 移到了终态组 → 分裂：$\Pi_2 = \{\{A,B,C\}, \{D\}, \{E\}\}$
  4. 再考察 $\{A,B,C\}$：$m(B,b)=D$（在 $\{D\}$ 组），而 $m(A,b)=C, m(C,b)=C$（仍在 $\{A,B,C\}$ 组）→ 分裂：$\Pi_3 = \{\{A,C\}, \{B\}, \{D\}, \{E\}\}$
  5. $\{A,C\}$ 不能再分裂：$m(A,a)=B, m(C,a)=B$（都在 $\{B\}$ 组）；$m(A,b)=C, m(C,b)=C$（都在 $\{A,C\}$ 组）
  6. 最终划分：$\Pi_{final} = \{\{A,C\}, \{B\}, \{D\}, \{E\}\}$
  7. 选代表：用 A 代表 $\{A,C\}$，修改状态转移
  8. 重命名：0=A, 1=B, 2=D, 3=E

最小化后的 DFA：

状态 a b
0 1 0
1 1 2
2 1 3
3 1 0

1.6 从 DFA 构造词法分析器

1.6.1 两种实现方式

方式一：直接编码的词法分析器 - 将 DFA 和 DFA 识别过程合并，直接用程序代码模拟 DFA 行为 - 状态与语句对应关系： - 初态 → 程序的开始 - 终态 → 程序的结束（不同终态 return 不同记号） - 状态转移 → 分情况或条件语句（case/if） - 环 → 循环语句（loop） - return 满足最长匹配原则

方式二：表驱动型的词法分析器 - DFA 是"被动的"数据（分析表），需要一个驱动器来模拟 DFA 的行为 - 结构：分析表 + 驱动器框架（驱动器与模式无关，是通用的）

1.6.2 词法分析器的自动生成工具 LEX

LEX 是贝尔实验室开发的词法分析器自动生成工具，已成为 UNIX 标准应用程序。

LEX 的基本结构： - 根据正规式构造的分析表（数据） - 驱动器框架（不变的程序逻辑）

利用 LEX 构造词法分析器的关键： 1. 用 LEX 提供的正规式集合设计记号的模式 2. 用 LEX 提供的语义支持识别记号或指出输入中的错误

1.6.3 两类分析器的比较

特性 直接编码 表驱动
实现方式 手工编码 自动生成
分析速度 快 较慢
灵活性 高 低
可维护性 差 好
适用场景 简单语言的编译器 复杂语言或使用工具

1.7 $\varepsilon$-闭包的计算（算法2.4补充）

计算 $\varepsilon$-闭包的算法：

function ε-闭包(T) is
begin
    for T 中每个状态 t loop
        加入 t 到 U; push(t);
    end loop;
    while 栈不空 loop
        pop(t);
        for 每个 u = move(t, ε) loop
            if u 不在 U 中 then
                加入 u 到 U; push(u);
            end if;
        end loop;
    end loop;
    return U;
end ε-闭包;

示例：计算 $\varepsilon\text{-闭包}(\{s_2\})$ - 步骤1：$U = \{s_2\}$，$stack = [s_2]$ - 步骤2：pop $s_2$，$move(s_2, \varepsilon) = s_4$，$U = \{s_2, s_4\}$，push $s_4$ - 步骤3：pop $s_4$，$move(s_4, \varepsilon) = s_5$，$U = \{s_2, s_4, s_5\}$，push $s_5$ - 步骤4：pop $s_5$，$move(s_5, \varepsilon)$ 为空 - 结果：$\varepsilon\text{-闭包}(\{s_2\}) = \{s_2, s_4, s_5\}$

二、相关真题与解析

题目1（来源：2022年 试题A）

原题： 已知正规式 $(b|a)^*a$ 描述的正规集，试给出： （1）使用 Thompson 算法构造识别该正规集的 NFA； （2）将 NFA 确定化为 DFA（要有计算过程）； （3）最小化的 DFA。

解析/答案：

(1) Thompson 算法构造 NFA

分解正规式：$(b|a)^*a$ - 先构造 $N(b)$ 和 $N(a)$ - 构造 $N(b|a)$（或运算） - 构造 $N((b|a)^*)$（闭包运算） - 与 $N(a)$ 连接

得到 NFA：包含状态集和相应的 $\varepsilon$ 转移。

(2) NFA 确定化为 DFA

  • 计算初态集的 $\varepsilon$-闭包
  • 对每个状态集和每个输入字符（a、b），计算 $U = \varepsilon\text{-闭包}(smove(T, ch))$
  • 将每个不同的 $U$ 作为一个新的 DFA 状态
  • 包含 NFA 终态的状态集即为 DFA 终态

(3) DFA 最小化 - 初始划分：非终态组与终态组 - 考察同一组中各状态对每个字符的转移是否进入同一组 - 若进入不同组则分裂 - 重复至不能再分裂

最终最小化 DFA 可用状态转移表表示。

题目2（来源：2022年 试题B）

原题： 已知正规式 $(a|b)^*ba$ 描述的正规集，试给出： （1）使用 Thompson 算法构造识别该正规集的 NFA； （2）将 NFA 确定化为 DFA（要有计算过程）； （3）最小化的 DFA。

解析/答案： 方法与上题完全一致，区别在于最后的两个字符是 "ba" 而非 "a"。在 Thompson 构造中，$(a|b)^*ba = ((a|b)^*)(b)(a)$，先构造 $N((a|b)^*)$，再依次与 $N(b)$、$N(a)$ 做连接运算。确定化和最小化的步骤相同。

题目3（来源：2022年 试题B）

原题： 简答题——能否使用正规式描述语言 $\{0^m1^m \mid m \geq 1\}$？原因是什么？

解析/答案： 不能。原因：语言 $\{0^m1^m \mid m \geq 1\}$ 不是正规语言，它不能用正规式描述。

根据泵引理（Pumping Lemma），正规语言必须满足泵性质。对于语言 $L = \{0^m1^m \mid m \geq 1\}$： - 该语言要求 0 的个数与 1 的个数严格相等，需要一个"计数器"来记住 0 出现了多少次 - 有限自动机的状态是有限的，无法"记住"任意数量的 0 - 这超出了正规语言的描述能力，需要用上下文无关文法（2型文法）来描述

对应的上下文无关文法：$S \to 0S1 \mid 01$。

题目4（来源：2022年 试题A）

原题： 填空题——一个 NFA 需要满足__________和__________，才是一个 DFA。

解析/答案： （1）没有 $\varepsilon$ 状态转移；（2）对每个状态 $s$ 和每个输入字符 $a$，最多有一个下一状态。这两个条件分别消除了 NFA 的 $\varepsilon$ 不确定性和状态转移不确定性。

题目5（来源：2022年 试题B）

原题： 填空题——LR(K)分析中，L表示__________，R表示__________，K表示__________。

解析/答案： L 表示从左到右扫描（Left-to-right scan），R 表示最右推导的逆（Rightmost derivation in reverse），K 表示向前看 K 个符号（lookahead symbols）。

题目6（来源：2022年 试题A）

原题： 填空题——产生式 $A \to \varepsilon$ 的 LR(0)项目有__________。

解析/答案： $A \to \cdot$（即只有圆点，没有后继符号）。因为 $A \to \varepsilon$ 的右部为空，按 LR(0)项目构造规则，该项目只有一个状态：圆点在产生式最前面（也是最后面），即 $A \to \cdot$。这是一个可归约项目。
