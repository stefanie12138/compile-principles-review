第四讲：语法制导翻译与中间代码生成

一、知识点讲解

1.1 语法制导翻译概述

1.1.1 语法与语义的关系

  • 语法：语言的结构，即语言的"样子" —— 是指语言结构是否正确的规则
  • 语义：附着于语言结构上的实际含义，即语言的"意义" —— 规定结构正确的句子所表示的意思是否合法

关键认识：

  • 语义不能离开语法独立存在
  • 语义远比语法复杂
  • 同一语言结构可包含多种含义，不同语言结构可表示相同含义
  • 语法与语义之间没有明确的界线

语义分析的两个作用：

  1. 检查结构正确的句子所表示的意思是否合法
  2. 执行规定的语义动作：表达式求值、符号表填写、中间代码生成等

1.1.2 语法制导翻译的基本思想

语法制导翻译（Syntax-Directed Translation, SDT）是处理语义的基本方法。它以语法分析为基础，伴随语法分析的各个步骤，执行相应的语义动作。

具体方法：

  • 将文法符号所代表的语言结构的意思，用附着于该文法符号的属性表示
  • 用语义规则规定产生式所代表的语言结构之间的关系（即属性之间的关系）
  • 在语法分析的适当时刻执行附着在对应产生式上的语义规则

1.2 属性文法

1.2.1 属性的定义

定义4.1：对于产生式 $A \to \alpha$（$\alpha = X_1X_2\ldots X_n$），其语义规则可以表示为关于属性的函数： $$b := f(c_1, c_2, \ldots, c_k)$$

属性的抽象表示形式：.attr 例如：$E.val$（值）、$E.type$（类型）、$E.code$（代码序列）、$E.place$（存储空间）

1.2.2 综合属性与继承属性

综合属性（Synthesized Attribute）： 若 $b$ 是 $A$ 的属性，$c_1, c_2, \ldots, c_k$ 是 $\alpha$ 中文法符号的属性或 $A$ 的其他属性，则称 $b$ 是 $A$ 的综合属性。

  • 综合属性自下而上计算（取决于孩子的属性）

继承属性（Inherited Attribute）： 若 $b$ 是 $\alpha$ 中某文法符号 $X_i$ 的属性，$c_1, c_2, \ldots, c_k$ 是 $A$ 的属性或 $\alpha$ 中其他文法符号的属性，则称 $b$ 是 $X_i$ 的继承属性。

  • 继承属性自上而下计算（取决于父亲的属性、兄弟的属性）

属性依赖关系：属性之间的依赖关系实质上反映了属性计算的先后次序——所有 $c_i$ 被计算之后才能计算属性 $b$。

1.2.3 语义规则的两种形式

形式 描述 适用阶段
语法制导定义 用抽象属性和运算符号表示的语义规则（做什么） 设计阶段
翻译方案 用具体属性和运算表示的语义规则（如何做） 实现阶段

二者作用等价——语法制导定义是算法，翻译方案是程序实现。

1.2.4 注释分析树

将属性附着在分析树对应文法符号上，形成注释分析树。从注释分析树可以看出：

  • 继承属性是自上而下计算的
  • 综合属性是自下而上计算的

1.2.5 LR分析中的语法制导翻译

LR分析中的语法制导翻译是对LR语法分析的扩充：

  1. 扩充LR分析器功能：当执行归约产生式的动作时，也执行产生式对应的语义动作。限制语义动作仅能放在产生式右部的最右边
  2. 扩充分析栈：增加一个与分析栈并列的语义栈，用于存放分析栈中文法符号所对应的属性值

示例：$E \to E_1 + E_2$，对应的语义动作为 val[top] := val[top] + val[top+2]

1.3 中间代码

1.3.1 中间代码的作用与特性

中间代码是编译器前端与后端的分水岭。一个好的中间代码应具备：

  • 便于语法制导翻译
  • 既与机器指令的结构相近，又与具体机器无关

1.3.2 后缀式（逆波兰表示）

操作数在前，操作符紧随其后，无需用括号限制运算的优先级和结合性。

例如：算术表达式 $3+5+8$ 的后缀式为 $3\ 5 + 8 +$

后缀式计算算法：操作数进栈，遇到算符时弹出操作数、计算、将结果进栈。最终结果留在栈中。

后缀式的推广：可推广到任何语句。如 if e then x else y 的后缀式可写为 e x y if-then-else。

1.3.3 三地址码

三地址码的直观形式：

  • result := arg1 op arg2（二元运算）
  • result := op arg1（一元运算）
  • op arg1

示例：$x := a + b * c$ 的三地址码序列：

T1 := b * c
T2 := a + T1
x := T2

三地址码的种类：

序号 三地址码 四元式
(1) $x := y\ op\ z$ $(op, y, z, x)$
(2) $x := op\ y$ $(op, y,\ , x)$
(3) $x := y$ $(:=, y,\ , x)$
(4) goto L $(j,\ ,\ , L)$
(5) if x goto L $(jnz, x,\ , L)$
(6) if x relop y goto L $(jrelop, x, y, L)$
(7) param x $(param,\ ,\ , x)$
(8) call n, P $(call, n,\ , P)$
(9) return y $(return,\ ,\ , y)$
(10) $x := y[i]$ $(=[], y[i],\ , x)$
(11) $x[i] := y$ $([]=, y,\ , x[i])$
(12) $x := \&y$ $(\&, y,\ , x)$
(13) $x := *y$ $(*, y,\ , x)$
(14) $*x := y$ $(*=, y,\ , x)$

1.3.4 三元式

三元式形式：$(i)\ (op, arg1, arg2)$，表示 $(i) := arg1\ op\ arg2$

特点：序号有双重含义——既代表此三元式，又代表三元式存放的结果。

弱点：给代码优化带来困难。因为删除或移动代码位置会导致序号变化，从而使其他三元式中对原序号的引用无效。

示例：$x := a + b * c$ 的三元式：

(1) (*, b, c)
(2) (+, a, (1))
(3) (:=, x, (2))

1.3.5 四元式

四元式是对三元式的改进：$(op, arg1, arg2, result)$，表示 $result := arg1\ op\ arg2$

关键改进：将三元式序号改为显式的临时变量，使得运算结果与其位置无关，为代码优化提供了极大方便。

示例：$x := a + b * c$ 的四元式：

(1) (*, b, c, T1)
(2) (+, a, T1, T2)
(3) (:=, x, T2, T3)

四元式的语法制导翻译要素：

  • 属性 .code：表示存放运算结果的变量
  • 函数 newtemp：返回一个新的临时变量
  • 过程 emit(op, arg1, arg2, result)：生成一个四元式

1.3.6 图形表示（树/DAG）

树作为中间代码：语法树真实反映句子结构，对语法树稍加修改（加入语义信息）即可作为中间代码。每个内部节点和它的孩子对应一个三元式或四元式。

DAG（有向无环图）：如果树上若干个节点有完全相同的孩子，则这些节点可以指向同一个孩子，实现资源共享。

树与其他中间代码的关系：

  • 对树进行深度优先后序遍历 → 后缀式
  • 树的每个内部节点和它的孩子 → 三元式或四元式
  • 现代的编译器基础架构均用语法树作为中间表示

1.4 符号表

1.4.1 符号表的作用与设计要求

符号表是连接声明与引用的桥梁，记住每个符号的相关信息（作用域、绑定等），帮助编译的各个阶段正确有效地工作。

设计要求：

  • 正确存储各类信息
  • 适应不同阶段的需求
  • 便于有效地进行查找、插入、删除和修改
  • 空间可以动态扩充

1.4.2 符号表的条目

每个声明的名字在符号表中占据一个条目，存放名字的相关信息。

  • 多个子表：变量名表、过程名表、保留字表等
  • 存放方式：关键字 + 属性
  • C 符号表的组合关键字至少包括：名字 + 作用域 + 类型

1.4.3 名字的作用域

名字的作用域：名字在哪个范围内起作用。

两种划分范围的方式：

  • 并列的：两个范围内的名字作用域互不相干
  • 嵌套的：需要用规则限定（如过程嵌套）

静态作用域规则（static-scope rule）：编译时就可以确定名字的作用域。

最近嵌套规则（most closely nested）：名字声明在离其最近的内层起作用——在名字引用处从内向外看，它处在所遇到的第一个该名字声明的作用域。

1.4.4 线性表组织

线性表应是一个栈结构，以正确反映名字的作用域。

基本操作：

  • 查找：从表头（栈顶）开始，遇到的第一个名字
  • 插入：先查找，再插入在表头
  • 删除：暂时删除（摘走并保存）或永久删除（摘走不再保存）
  • 修改：与查找类似（可用删除+插入代替）

效率（n个条目）：

  • 成功查找（平均）：$(n+1)/2$
  • 不成功查找：$n+1$
  • 建立 n 个条目的符号表（最坏）：$\sum i = (n+1)n/2$

1.5 声明语句的翻译

1.5.1 变量的类型定义与声明

  • 类型定义：为编译器提供存储空间大小的信息
  • 变量声明：为变量分配存储空间

简单数据类型的存储空间是预定义的（如 integer=4字节，real=8字节，char=1字节），组合数据类型的存储空间需要编译器根据程序员提供的信息计算。

1.5.2 变量声明的语法制导翻译

文法：

D → D ; D | id : T
T → int | real | array [num] of T | ^T

翻译要素：

  • 全程量 offset：记录当前符号存储的偏移量，初值设为 0
  • 属性 .type 和 .width：变量的类型和占据的存储空间
  • 过程 enter(name, type, offset)：建立符号表条目并分配存储空间

语义规则：

D → id : T    {enter(id.name, T.type, offset); offset := offset + T.width}
T → int       {T.type := integer; T.width := 4}
T → real      {T.type := real; T.width := 8}
T → array[num] of T1  {T.type := array(num.val, T1.type); T.width := num.val * T1.width}
T → ^T1       {T.type := pointer(T1.type); T.width := 4}

1.5.3 过程的定义与声明

过程的三种形式：过程定义、过程声明、过程调用。

先声明后引用原则：过程定义可以写在对它的引用之后。引用前必须先声明。若引用前已定义，则声明可省略。

1.5.4 左值与右值

  • 左值（lvalue）：具有存储空间，是"容器"
  • 右值（rvalue）：可以仅是一个值，没有存储空间，是"内容"

出现在赋值号左边的变量称为左值，出现在右边的称为右值。常量、字面量、函数返回值、表达式的值不能作为左值。

1.5.5 参数传递方式

传递方式 特点 实参要求 对实参的影响
值调用 传递实参的右值（副本） 可以是右值 不影响实参原值
引用调用 传递实参的地址 必须是左值 修改形参即修改实参
复写-恢复 值调用+返回时恢复 必须是左值 返回时恢复，避免副作用
换名调用 宏替换方式展开 任意正文 直接文本替换

C语言仅支持值调用，可通过传递指针模拟引用调用的效果。

引用调用的副作用：当实参与非本地量共用存储空间时，过程内修改参数值的同时也改变了非本地量的值。

1.5.6 嵌套过程符号表的作用域信息

在允许嵌套定义过程的语言中，符号表使用有嵌套结构的组织方式：

  • 每个过程被认为是一个子符号表（节点）
  • 嵌套的节点之间用双向链连接
  • 正向链指示过程的嵌套关系
  • 逆向链实现按作用域对名字进行访问

定义4.2：设主程序的嵌套深度 $d_{main} = 1$，若过程 $A$ 直接嵌套定义过程 $B$，则 $d_B = d_A + 1$。

语义函数：

  • mktable(previous)：建立新节点，返回指针
  • enter(table, name, type, offset)：在节点中建立新条目
  • addwidth(table, width)：计算节点中所有条目的累加宽度
  • enterproc(table, name, newtable)：为过程名建立条目

1.6 可执行语句的翻译

1.6.1 简单算术表达式与赋值句

文法：$A \to id := E$，$E \to E+E \mid E*E \mid -E \mid (E) \mid id$

翻译要素：

  • 属性 .place：存放 $E$ 的变量名地址
  • 过程 emit()：生成三地址码

语义规则：

A → id := E    {emit(entry(id.name) ':=' E.place)}
E → E1 + E2    {E.place := newtemp; emit(E.place ':=' E1.place '+' E2.place)}
E → E1 * E2    {E.place := newtemp; emit(E.place ':=' E1.place '*' E2.place)}
E → -E1        {E.place := newtemp; emit(E.place ':=' '-' E1.place)}
E → (E1)       {E.place := E1.place}
E → id         {E.place := entry(id.name)}

1.6.2 类型转换

强制转换（coercion）：按照一定原则，将不同类型的变量在内部转换为相同的类型。

  • T := itr E：将 $E$ 从整型变为实型
  • T := rti E：将 $E$ 从实型变为整型

语义规则需根据左右部类型是否相同判断是否需要插入类型转换代码。

1.6.3 布尔表达式的翻译

布尔表达式的两种计算方法：

方法一：数值表示的直接计算

  • 用 1 表示 true，0 表示 false
  • or、and、not 与 +、*、-（一元）对应
  • 关系运算通过条件跳转实现

方法二：逻辑表示的短路计算

  • $A\ or\ B$：if $A$ then true else $B$
  • $A\ and\ B$：if $A$ then $B$ else false
  • $not\ A$：if $A$ then false else true

短路计算的必要性：可以避免运行时错误。例如：

while ptr<>nil and ptr^.data=x do ...

短路计算可以回避对 ptr^.data=x 的判断。

短路计算的语法制导定义：

  • 属性 .true：表达式的真出口
  • 属性 .false：表达式的假出口
  • 函数 newlabel：产生新标号

拉链与回填：

  • 当三地址码中的转向不确定时，将所有转向同一地址的三地址码拉成一个链
  • 一旦所转向的地址被确定，则沿此链回填地址

1.6.4 控制流语句的翻译

控制流语句（if-then-else、while-do 等）的翻译需要使用拉链-回填技术处理布尔表达式的短路计算，并正确生成条件跳转和无条件跳转的三地址码。

二、相关真题与解析

题目1（来源：2022年 试题A）

原题： 填空题——属性文法中终结符具有__________属性，它的属性值由__________提供。

解析/答案： 综合属性；由词法分析器（或词法分析程序）提供。终结符（如 id、num）的属性通常是综合属性，其值由词法分析阶段确定（如 id.name、num.lexval）。

题目2（来源：2022年 试题A）

原题： 填空题——三元式中序号的作用是__________和__________。

解析/答案： （1）标识该三元式；（2）存放三元式的计算结果。三元式的序号具有双重含义——既代表该三元式本身，又代表该三元式运算结果存放的位置。这一特点导致代码优化困难。

题目3（来源：2022年 试题A）

原题： 填空题——注释语法树中文法符号的继承属性可以由__________结点的属性和__________结点的属性来计算。

解析/答案： 父结点的属性和兄弟（或左兄弟）结点的属性。继承属性是自上而下计算的——取决于父亲和兄弟（通常是左边兄弟）的属性值。与之相对，综合属性是自下而上计算的，取决于孩子的属性。

题目4（来源：2022年 试题A）

原题： 填空题——对抽象语法树进行__________可以得到后缀式形式的中间代码。

解析/答案： 深度优先后序遍历。对抽象语法树进行深度优先的后序遍历，得到的线性序列就是后缀式（逆波兰表示）。每个内部节点和它的孩子对应一个三元式或四元式。

题目5（来源：2022年 试题A）

原题： 简答题——规定逻辑运算符的优先级从高到低为 $\neg$、$\land$、$\lor$、$\to$，并且 $\neg$ 和 $\lor$ 具有右结合性，$\land$ 和 $\to$ 具有左结合性。给出逆波兰式 $AB \to \neg\neg C \lor D \lor E \land$ 的中缀式表示。

解析/答案： 从逆波兰式出发，根据运算符的结合性和优先级还原中缀式。

逆波兰式（后缀式）的求值过程：遇到操作数进栈，遇到运算符弹出操作数形成子表达式。

逆波兰式 $A\ B\ \to\ \neg\ \neg\ C\ \lor\ D\ \lor\ E\ \land$：

  • 读入 $A, B$ → 栈中：$A, B$
  • 遇到 $\to$：$(A \to B)$
  • 遇到 $\neg$：$\neg(A \to B)$
  • 遇到 $\neg$：$\neg\neg(A \to B)$
  • 读入 $C$：$\neg\neg(A \to B), C$
  • 遇到 $\lor$：$(\neg\neg(A \to B) \lor C)$（$\lor$ 右结合）
  • 读入 $D$：$(\neg\neg(A \to B) \lor C), D$
  • 遇到 $\lor$：$(\neg\neg(A \to B) \lor (C \lor D))$
  • 读入 $E$：$(\neg\neg(A \to B) \lor (C \lor D)), E$
  • 遇到 $\land$：$((\neg\neg(A \to B) \lor (C \lor D)) \land E)$

中缀式：$(\neg\neg(A \to B) \lor (C \lor D)) \land E$

题目6（来源：2022年 试题A）

原题： 简答题——某表达式计算的语法制导定义如下，画出表达式 $15+(6*7+8)$ 的注释语法树。

L → E { print(E.val) }
E → E1 * T { E.val := E1.val + T.val }
E → T { E.val := T.val }
T → T1 + F { T.val := T1.val - F.val }
T → F { T.val := F.val }
F → (E) { F.val := E.val }
F → num { F.val := num.lexval }

解析/答案： 注意此题中运算符语义被"交换"了（* 对应加法、+ 对应减法）。

表达式 $15+(6*7+8)$ 的语法树（按给定优先级，+ 优先级高于 *，但括号改变优先级）：

  1. 先计算括号内 $6*7+8$：
  2. $6*7$ 对应 $T \to T1+F$，所以 $T.val = 6-7 = -1$
  3. $-1+8$ 对应 $E \to E1*T$，所以 $E.val = (-1)+8 = 7$
  4. 括号内值为 $7$
  5. 计算 $15+7$：
  6. 对应 $E \to E1*T$，所以 $E.val = 15+7 = 22$

最终 $E.val = 22$，输出 $22$。

注释语法树中每个结点标注对应的 .val 属性值。

题目7（来源：2022年 试题A）

原题： 计算题——将下列语句翻译成三地址码（布尔表达式要求采用短路计算）：

while not a>b and c<d
do
    while c<e
    do
        if d>e
        then e:=b*d
        else c:=a+b

其中 a、b 为整型，c、d、e 为实型。

解析/答案： 使用短路计算翻译，需引入标号表示真假出口，并正确处理类型转换。

L1: if a>b goto L2         -- not a>b 为假时跳转
    goto L3                -- not a>b 为真
L2: if c<d goto L4         -- c<d 为真时进入循环体
    goto L_next             -- 整个条件为假，退出外层循环
L3: if c<d goto L4
    goto L_next
L4: if c<e goto L5         -- 内层 while 条件
    goto L1                -- 内层条件为假，返回外层
L5: if d>e goto L6         -- if 条件
    goto L7
L6: t1 := itr b             -- b 整型转实型
    t2 := t1 * d
    e := t2
    goto L4
L7: t3 := itr a             -- a 整型转实型
    t4 := t3 + b            -- 注意：a+b 中 a 转实型
    c := rti t4             -- 转回整型(若 c 为整型)或直接赋值
    goto L4
L_next:                     -- 循环出口

题目8（来源：2022年 试题B）

原题： 填空题——名字的作用域规则包括__________原则和__________原则。

解析/答案： 静态作用域原则和最近嵌套原则。静态作用域规则意味着编译时就可以确定名字的作用域；最近嵌套规则意味着名字声明在离其最近的内层起作用。

题目9（来源：2022年 试题B）

原题： 填空题——参数传递形式有__________、__________、__________、换名调用。

解析/答案： 值调用（call by value）、引用调用（call by reference）、复写-恢复（copy-in/copy-out）。这是四种常见的参数传递形式。

题目10（来源：2022年 试题B）

原题： 填空题——语义规则的两种表现方式是__________和__________。

解析/答案： 语法制导定义和翻译方案。语法制导定义用抽象属性和运算符号表示（做什么），翻译方案用具体属性和运算表示（如何做）。二者作用等价。

题目11（来源：2022年 试题B）

原题： 简答题——四元式与三元式的区别是什么？有什么优点？

解析/答案： 区别：三元式用序号来表示运算结果，三元式的值与其在三元式表中的位置相关；四元式将运算结果改用显式的临时变量来表示。

四元式的优点：运算结果与四元式的位置无关，为代码优化提供了极大方便——可以自由地删除或移动四元式而不影响运算结果（因为临时变量名不依赖于位置）。三元式的弱点是一旦进行代码的删除或移动，序号会发生变化，导致其他三元式中对原序号的引用失效。
